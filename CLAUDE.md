# Project Rules

## Validation First — No Assumptions

- Always validate claims by checking the actual code, database, or data before stating anything as fact.
- Never assume a feature works, a value exists, or data is correct — verify it first.
- If something cannot be verified, clearly state that it is unverified or unknown.
- When reporting numbers (counts, prices, margins), query the real source — do not estimate or guess.
- If unsure, ask rather than assume.
